SecureVault Identity System
==================================
Laboratory Exercise Description

This laboratory exercise is about creating a secure identity system using Flutter. In this project, we applied the Strict MVVM (Model-View-ViewModel) architecture to properly separate the user interface, business logic, and services. The main goal of this activity is to understand how authentication systems work and how to implement them securely in a mobile application.

In this system, users can register and login using email and password. We also implemented Google Sign-In as an alternative login method. For additional security, we integrated biometric authentication such as fingerprint login. Instead of using normal local storage, we used secure storage to safely store authentication tokens.

Through this laboratory exercise, we learned how to organize a Flutter project properly, apply clean architecture principles, handle authentication securely, and collaborate as a team using assigned roles. This activity helped us understand the importance of security, structure, and teamwork in real-world mobile application development.
